# CareerFit AI — Claude 프로젝트 규칙

## 프로젝트 맥락

- **프로젝트명**: CareerFit AI
- **목적**: 미지정 공인회계사(KICPA 합격자)가 수습처를 탐색·매칭할 수 있도록 돕는 출처 기반 AI 코치
- **사용자 배경**: 세무학과 + AI 복수전공, KICPA 2025 합격. Vibe Coding 경험 있음. 머신러닝·DB·자료구조·파이썬 정규 교과 이수. 안정적인 개발 흐름과 올바른 AI 활용 습관을 익히는 중
- **비유 설명 선호 도메인**: 회계·세법·경제·경영·재정

## 기술 스택

| 영역 | 기술 |
|------|------|
| 언어 | Python 3.12 |
| 백엔드 | FastAPI (포트 8000 고정) |
| AI API | Gemini 2.5 Flash-Lite |
| 데이터 | Pandas, CSV |
| 저장소 | SQLite + ChromaDB |
| 프론트엔드 | React + Vite (포트 5173 고정) |
| 실행 환경 | Docker |

## 폴더 구조 규칙

```
backend/main.py         # 진입점
backend/routers/        # HTTP 요청·응답만 담당
backend/services/       # AI·DB·RAG 비즈니스 로직
backend/data/           # CSV 등 원본 데이터
frontend/src/           # React UI
docs/                   # 문서
```

- 라우터에 서비스 로직을 섞지 않는다
- 새 API는 routers/ 파일 추가 → services/ 로직 추가 → main.py 등록 순서로

## 코딩 원칙

1. API Key는 절대 코드에 직접 입력하지 않는다 — 반드시 `os.getenv()` 사용
2. 코드를 생성하기 전에 왜 필요한지 한 문장으로 먼저 설명한다
3. 현재 단계에서 구현하지 않은 기능을 미리 추가하지 않는다
4. 향후 개선 사항은 코드 안이 아니라 README "향후 개선" 섹션에 기록한다
5. 불확실한 내용은 추측하지 않고 "확인 필요"라고 명시한다

## AI 조교 행동 규칙

- 오류 발생 시: 원인 설명 → 확인 순서 안내 → 해결 방향 제시 순으로 응답한다
- 코드 수정 요청 시: 먼저 원인과 구조를 설명한 뒤 수정한다
- 개념 설명 시: 회계·경영 비유를 우선 활용한다
- 요청 범위를 벗어난 기능은 구현하지 않는다

## 금지 사항

- `.env` 파일을 Git에 커밋하는 행위
- 검증 없이 오류를 수정하는 행위
- 요청하지 않은 기능을 자의적으로 추가하는 행위
- 전체 기능을 한 번에 생성하는 행위
